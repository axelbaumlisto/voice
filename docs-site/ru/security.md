# Безопасность

Не коммитьте API-ключи, токены доступа, локальные базы SQLite, определения провайдеров, промпты, файлы failed-audio, логи, debug-аудио и сгенерированные пользовательские данные.

Учётные данные должны оставаться локальными и настраиваться через UI настроек / локальное хранилище конфигурации. Приложение **не** загружает API-ключи автоматически из переменных окружения.

Локальные runtime-данные включают `config.db`, `history.db`, `dictionary.txt`, `corrections.db`, `providers.db`, `prompts.db`, `failed_audio/`, `debug/`, `logs/` и `themes/` в каталоге конфигурации платформы.

## Граница доверия тем

Темы оверлея — это исполняемый JavaScript в webview приложения. Они доверенные по дизайну и **не** являются песочницей. Устанавливайте темы только из источников, которым доверяете, и проверяйте `theme.js` перед использованием сторонних тем.

- Включённый режим отладки (`debug=true`) сохраняет WAV и JSONL в `debug/` без автоматической очистки, что может увеличить расход диска или раскрыть записи. Отключайте debug при обычном использовании.

Перед публикацией просканируйте репозиторий на реальные секреты.

```bash
git grep -InE '(ghp_|gho_|github_pat_|sk-[A-Za-z0-9]|GROQ_API_KEY|OPENAI_API_KEY|ANTHROPIC_API_KEY|access[_-]?token|api[_-]?key|secret)' -- . \
  ':(exclude)bun.lock' ':(exclude)bun.lockb'
```

Если реальный секрет когда-либо попал в публичный репозиторий, немедленно отзовите/смените его. Удаления из текущего дерева недостаточно, если старая Git-история публична.
